### Week 3 Achievements:
- Developed a solid understanding of AWS networking fundamentals, especially the structure and purpose of Amazon VPC in cloud architecture.
- Learned key VPC concepts such as CIDR blocks, public and private subnets, route tables, and the relationship between different networking components.
- Successfully created and configured a custom VPC environment, which helped strengthen practical understanding of network segmentation in AWS.
- Gained a clearer understanding of how route tables control traffic flow between subnets and external destinations.
- Studied Security Groups and Network ACLs and understood the differences between stateful and stateless traffic filtering mechanisms.
- Practiced configuring inbound and outbound access rules, which improved understanding of how traffic can be controlled and restricted at different layers of the network.
- Learned how Internet Gateway and NAT Gateway support internet connectivity for public and private resources in a VPC.
- Built a simple multi-tier architecture with separate web, application, and database layers, which provided practical experience in organizing cloud resources based on security and access requirements.
- Strengthened overall understanding of AWS network design principles, especially in the areas of segmentation, controlled access, and secure cloud architecture.
