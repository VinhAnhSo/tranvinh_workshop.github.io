### Week 4 Achievements:
- Gained a solid understanding of AWS IAM core concepts, including IAM Users, IAM Groups, IAM Roles, and IAM Policies.
- Learned the difference between authentication and authorization, which helped clarify how AWS controls identity verification and permission management.
- Practiced reading and writing IAM policy documents in JSON format and improved understanding of how permissions are structured in AWS.
- Understood the distinction between managed policies and inline policies, along with the use cases for each approach in access control design.
- Successfully configured MFA for account protection and strengthened awareness of multi-factor authentication as an important AWS security best practice.
- Applied password policy settings and explored permission boundaries to better understand how user access can be restricted and controlled.
- Built a simple secure multi-user environment and gained practical experience in assigning permissions based on roles and responsibilities.
- Improved overall awareness of the least-privilege principle and the importance of security-focused IAM design in AWS environments.
