- Understand AWS storage and database services, including S3, EBS, and RDS.

### Week 2 Achievements:
- Developed a solid understanding of AWS storage and database services, especially the roles of Amazon S3, EBS, and RDS in cloud application design.
- Learned how Amazon S3 provides highly durable object storage and gained practical knowledge of buckets, objects, and access control considerations.
- Explored S3 Lifecycle policies and understood how automated data transition and deletion can support better storage management.
- Studied static website hosting with Amazon S3 and became aware of modern alternatives such as AWS Amplify Hosting for more advanced deployment scenarios.
- Gained knowledge of Amazon EBS volume characteristics, including their relationship to Availability Zones, resizing options, and flexible volume type changes.
- Learned how EBS snapshots work as point-in-time backups and understood their role in backup, recovery, encryption, and storage protection.
- Explored Amazon RDS architecture and understood the benefits of managed relational databases, including automated maintenance, patching, and support for multiple database engines.
- Improved understanding of secure cloud architecture by learning how database services such as RDS are typically placed in private subnets while remaining accessible to application components.
- Successfully built a simple file upload system that integrated EC2, S3, and RDS, which strengthened practical understanding of how multiple AWS services can work together in a functional application.
