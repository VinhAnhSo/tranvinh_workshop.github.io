## Nội dung chính

### Phiên 1: Elixir trong DevOps
- Ngôn ngữ lập trình functional chạy trên BEAM VM  
- Hỗ trợ concurrency cao với lightweight process  
- Fault-tolerance nhờ OTP và supervision tree  
- Triết lý “Let it crash” giúp hệ thống tự phục hồi  

### Phiên 2: Kubernetes
- Điều phối container ở quy mô lớn  
- Kiến trúc gồm Control Plane và Worker Node  
- Thành phần chính: Pod, Deployment, ReplicaSet  
- Tính năng: auto scaling, self-healing, rolling update  

### Phiên 3: Infrastructure as Code
- Quản lý hạ tầng bằng code thay vì thao tác thủ công  
- Công cụ: CloudFormation, CDK, Terraform  
- Lợi ích: tự động hóa, nhất quán, dễ tái sử dụng  
- Quy trình: định nghĩa → triển khai → quản lý  

## Bài học rút ra

1. DevOps cần tự động hóa và khả năng mở rộng  
2. Kubernetes là tiêu chuẩn container orchestration  
3. IaC giúp giảm lỗi thủ công và tăng tính ổn định  
4. Elixir phù hợp cho hệ thống xử lý đồng thời cao  

## Ứng dụng thực tế

- Sử dụng Terraform/CDK để triển khai hạ tầng cloud  
- Áp dụng Kubernetes để scale backend hoặc hệ thống AI  
- Sử dụng Elixir cho hệ thống real-time hoặc phân tán
