## Event Objectives
- Understand DevOps and cloud-native architecture  
- Learn Kubernetes for scalable systems  
- Explore Infrastructure as Code (IaC)  
- Discover Elixir for concurrent and fault-tolerant systems  

## Speakers
- **Nguyen Ta Minh Triet** – Elixir in DevOps  
- **Bao Huynh** – Kubernetes Architecture  
- **Nguyen Thinh** – Infrastructure as Code  

## Key Highlights

### Session 1: Elixir in DevOps
- Functional programming language running on BEAM VM  
- Supports high concurrency with lightweight processes  
- Fault tolerance using OTP and supervision trees  
- “Let it crash” philosophy for system recovery  

### Session 2: Kubernetes
- Container orchestration for large-scale systems  
- Architecture includes Control Plane and Worker Nodes  
- Core objects: Pod, Deployment, ReplicaSet  
- Features: auto scaling, self-healing, rolling updates  

### Session 3: Infrastructure as Code
- Manage infrastructure using code instead of manual operations  
- Tools: AWS CloudFormation, CDK, Terraform  
- Benefits: automation, consistency, reproducibility  
- Workflow: define → deploy → manage  

## Key Takeaways
1. DevOps requires automation and scalability  
2. Kubernetes is the standard for container orchestration  
3. IaC reduces human error and improves consistency  
4. Elixir is suitable for highly concurrent systems  

## Applying to Real Projects
- Use Terraform or CDK to deploy cloud infrastructure  
- Apply Kubernetes to scale backend or AI systems  
- Use Elixir for real-time or distributed processing systems
